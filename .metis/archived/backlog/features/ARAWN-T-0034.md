## Objective
Gate tool execution behind user approval. Modes: `default` (prompt for dangerous tools), `auto` (approve all), `read-only` (block writes/shell). Check permissions before each `Tool::execute`. Denied tools return error to LLM with "permission denied" message. Claude Code has 4 modes + organization MDM override — start with default + auto. Requires TUI for interactive prompting; in headless/CLI mode, use `auto` or config-based allow lists.

### Priority
- P1 — critical for safety but requires TUI for interactive approval flow
