# Review and document default agent configuration values

## Objective

Audit all default configuration values for the agent and ensure they're sensible. Current defaults may be too restrictive or undocumented. The orchestrator uses `max_iterations=1` by design (single-step turns), but this produces noisy "Max iterations exceeded" warnings in logs that look like errors.

### Type
- [x] Tech Debt - Code improvement or refactoring

### Priority
- [x] P1 - High (important for user experience)

### Current Defaults (arawn-types defaults module)
- `MAX_ITERATIONS`: 25 (main agent), 1 (orchestrator sub-agents)
- `max_tokens`: 65536 (from config.toml)
- `shell_timeout`, `web_timeout`, `max_output_bytes`: need audit

### Known Issues
- Orchestrator agents use `max_iterations=1` by design (check context size after every tool call for compaction). This is correct, but produces WARN-level "Max iterations exceeded" logs that look like errors — should be DEBUG since it's expected behavior in the orchestrator path
- No documentation of what each config value does and what reasonable ranges are
- Plugin-spawned agents inherit `default_max_iterations` from `[agent.default]` config — unclear if this is always desirable

## Acceptance Criteria

- [ ] All default values reviewed and confirmed sensible for typical usage
- [ ] Orchestrator's `max_iterations=1` pattern doesn't produce misleading WARN logs
- [ ] Config values documented (inline comments in defaults module, or config reference doc)
- [ ] Default config.toml generated by `arawn init` includes commented-out options with explanations
